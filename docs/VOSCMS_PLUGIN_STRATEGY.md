# VosCMS 플러그인 분리 전략

> 작성일: 2026-04-05
> VosCMS 코어에서 업종별 기능을 플러그인으로 분리하는 전략 문서

---

## 1. 현재 상태

### 이미 플러그인으로 분리 완료
| 플러그인 | 설명 | 상태 |
|----------|------|------|
| vos-pos | POS 시스템 | ✅ 활성 |
| vos-kiosk | 키오스크 | ✅ 활성 |
| vos-attendance | 근태 관리 | ✅ 활성 |

### 아직 코어에 남아있는 기능
| 기능 | 설명 |
|------|------|
| 예약 관리 | 예약 CRUD, 캘린더, 통계 |
| 서비스 관리 | 서비스 목록, 가격, 카테고리 |
| 스태프 관리 | 직원 목록, 스킬, 배정 |
| 묶음 서비스 | 번들 서비스 관리 |

### VosCMS 코어 (분리 후 남는 것)
- 페이지 관리
- 메뉴 관리
- 위젯 빌더
- 게시판 시스템
- 회원 관리
- 다국어 시스템
- 스킨/레이아웃
- 설정 관리

## 2. 분리 전략 비교

### A안: 개별 플러그인 분리

각 기능을 독립 플러그인으로 분리

```
vos-service      ← 독립 (의존 없음)
vos-staff        ← 독립 (의존 없음)
vos-reservation  ← vos-service 필수, vos-staff 선택
vos-pos          ← vos-reservation 선택
vos-kiosk        ← vos-reservation 필수
vos-attendance   ← vos-staff 필수
```

**장점:**
- 자유로운 조합 가능
- 업종별 맞춤 구성
- 마켓플레이스에서 개별 판매 가능

**단점:**
- 작업량 3배
- 의존성 관리 복잡
- 선택적 기능 분기 처리 필요
- 마켓플레이스 없는 현재 시점에서 과도한 투자

---

### B안: 업종별 번들 (채택)

관련 기능을 업종 단위로 묶어서 분리

```
vos-salon/        ← 살롱 번들 (예약+서비스+스태프+묶음서비스)
  plugin.json
  views/
    reservations/
    services/
    staff/
    bundles/
```

**장점:**
- 작업량 적음 (기존 코드 이동)
- 설치 한 번으로 즉시 운영 가능
- 외부 서버 테스트 바로 가능
- 업종별 용어/워크플로우 커스텀 가능

**단점:**
- 업종 간 코드 중복 가능성
- 나중에 개별 분리 시 재작업 필요

---

## 3. 업종별 번들 구성

### 3-1. 살롱/미용실 번들 (vos-salon)

미용실, 네일샵, 에스테틱, 마사지 등 뷰티 업종

**포함 기능:**
- 서비스 관리 (시술 메뉴, 가격, 소요시간)
- 스태프 관리 (디자이너, 스킬, 스케줄)
- 예약 관리 (예약, 캘린더, 통계)
- 묶음 서비스 (세트 메뉴)

**의존 플러그인 (별도 설치):**
- vos-pos (선택) — 현장 결제
- vos-kiosk (선택) — 셀프 체크인
- vos-attendance (선택) — 직원 근태

### 3-2. 향후 번들 후보

| 번들 | 대상 업종 | 핵심 차이점 |
|------|----------|------------|
| vos-salon | 미용실, 네일, 에스테틱 | 스태프별 시술, 소요시간 기반 예약 |
| vos-clinic | 병원, 클리닉 | 진료과, 의사 배정, 차트 연동 |
| vos-restaurant | 레스토랑, 카페 | 테이블, 좌석, 메뉴, 주문 |
| vos-academy | 학원, 교육 | 강좌, 강사, 수강생, 시간표 |
| vos-rental | 렌탈, 대여 | 장비/공간, 대여기간, 반납 |
| vos-fitness | 헬스장, 요가 | 수업, 정원, 회원권, 출석 |

## 4. 의존성 구조

```
VosCMS 코어
  │
  ├── vos-salon (번들)
  │     ├── 서비스 관리
  │     ├── 스태프 관리
  │     ├── 예약 관리
  │     └── 묶음 서비스
  │
  ├── vos-pos (독립, salon 선택 의존)
  │     └── salon 없으면 단순 판매 모드
  │
  ├── vos-kiosk (salon 필수 의존)
  │     └── 예약+서비스+스태프 필요
  │
  └── vos-attendance (독립)
        └── 스태프 데이터 참조만
```

## 5. 외부 서버 테스트 계획

### 테스트 시나리오

**시나리오 1: VosCMS만 설치**
- 플러그인 0개
- 페이지, 메뉴, 게시판, 회원 관리만 동작
- 일반 홈페이지로 사용 가능한지 확인

**시나리오 2: VosCMS + vos-salon**
- 살롱 번들 설치
- 서비스/스태프/예약 관리 동작 확인
- POS/키오스크 없이도 예약 접수 가능한지

**시나리오 3: VosCMS + vos-salon + vos-pos + vos-kiosk**
- 전체 설치 (= RezlyX 동일)
- 기존 RezlyX와 동일하게 동작하는지

**시나리오 4: VosCMS + vos-pos만**
- 예약 없이 POS만 사용
- 소매점처럼 단순 판매만 가능한지

### 테스트 전 필요 작업

1. ~~POS/키오스크/근태 플러그인 분리~~ (완료)
2. 예약/서비스/스태프 → vos-salon 번들 분리
3. VosCMS 설치 스크립트 작성 (DB 초기화 + 기본 설정)
4. 외부 서버 준비 (클린 환경)

## 6. 작업 순서

### Step 1: vos-salon 번들 분리 (개발 서버)
- 예약/서비스/스태프/번들 뷰 파일 → plugins/vos-salon/views/
- plugin.json 작성 (라우트, 메뉴, 마이그레이션)
- index.php에서 하드코딩 라우트 제거
- 사이드바 하드코딩 메뉴 제거
- 동작 테스트

### Step 2: VosCMS 설치 스크립트
- DB 마이그레이션 자동 실행
- 기본 관리자 계정 생성
- 기본 설정값 초기화
- 기본 메뉴/페이지 생성

### Step 3: 외부 서버 클린 설치 테스트
- VosCMS 코어만 설치 → 동작 확인
- 플러그인 추가 설치 → 동작 확인
- 조합별 시나리오 테스트

### Step 4: RezlyX 패키징
- VosCMS + 살롱 번들 + POS + 키오스크 + 근태 = RezlyX
- 원클릭 설치 패키지

## 7. 판매 모델

```
VosCMS (무료/오픈소스)
  │
  ├── vos-salon 번들 (유료)     → 미용실/살롱
  ├── vos-clinic 번들 (유료)    → 병원/클리닉
  ├── vos-restaurant 번들 (유료) → 레스토랑
  │
  ├── vos-pos (유료)            → 업종 공통 추가 모듈
  ├── vos-kiosk (유료)          → 업종 공통 추가 모듈
  ├── vos-attendance (유료)     → 업종 공통 추가 모듈
  │
  ├── 프리미엄 테마 (유료)
  │
  └── RezlyX 번들 (유료 올인원)
        = VosCMS + salon + pos + kiosk + attendance
```
