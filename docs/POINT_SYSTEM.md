# RezlyX 적립금 & 포인트 시스템

## 개요

RezlyX의 적립금/포인트 시스템은 **3가지 독립 필드**로 운영됩니다.

| 필드 | 이름 | 설명 | 변동 |
|------|------|------|------|
| `point` | 사용 가능 포인트 | 게시판 활동으로 적립. 환전하여 적립금으로 변환 가능 | 증감 |
| `balance` | 적립금 잔액 | 결제 시 자동 적립 + 포인트 환전으로 충전. 시술비 결제에 사용 | 증감 |
| `total_accumulated` | 누적 포인트 | 레벨 산정 기준. 포인트/적립금 발생 시 가중치 적용하여 합산 | **증가만** |

---

## 핵심 규칙

### 1. 포인트 (point) — 활동 보상

게시판 활동(글 작성, 댓글, 추천, 파일 업로드 등)을 통해 적립됩니다.

- 사용 가능 포인트에 직접 적립
- 환전 기능을 통해 적립금(balance)으로 변환 가능
- 환전 시 포인트 차감, 적립금 증가
- 삭제 시 회수 가능 (설정에 따라)

**예시: 미용실 고객 A씨의 게시판 활동**

| 활동 | point 변동 | point 잔액 | total_accumulated |
|------|-----------|-----------|-------------------|
| 초기 상태 | - | 0 | 0 |
| 시술 후기 작성 (텍스트) | +500 | 500 | +500 → 500 |
| 후기에 사진 첨부 | +200 | 700 | +200 → 700 |
| 다른 회원이 내 글 추천 | +100 | 800 | +100 → 800 |
| 다른 회원 글에 댓글 | +50 | 850 | +50 → 850 |
| 1,000pt 환전 신청 | - | (아래 환전 참고) | 변동 없음 |

### 2. 적립금 (balance) — 결제 보상

서비스 결제 시 자동 적립됩니다. 실제 금액(원/엔 등)과 동일한 단위입니다.

- 결제 금액의 N% 자동 적립 (회원 등급별 적립률 차등)
- 포인트 환전으로도 충전 가능
- 다음 시술 결제 시 차감 사용 가능
- 관리자 수동 지급/차감 가능

**예시: A씨의 적립금 흐름**

| 행동 | balance 변동 | balance 잔액 | total_accumulated |
|------|-------------|-------------|-------------------|
| 초기 상태 | - | 0 | 850 (위 활동에서) |
| 커트 50,000원 결제 (적립률 3%) | +1,500원 | 1,500원 | +1,500×가중치3 = +4,500 → 5,350 |
| 염색 80,000원 결제 (적립률 3%) | +2,400원 | 3,900원 | +2,400×3 = +7,200 → 12,550 |
| 포인트 850pt 환전 (비율 1pt=1원) | +850원 | 4,750원 | 변동 없음 |
| 다음 시술에 적립금 2,000원 사용 | -2,000원 | 2,750원 | 변동 없음 |

### 3. 누적 포인트 (total_accumulated) — 레벨 기준

**절대 줄어들지 않는** 누적 수치입니다. 레벨과 회원 등급 산정의 기준이 됩니다.

**합산 규칙:**
- 게시판 활동 포인트 적립 시: `적립 포인트 × 활동 가중치` 합산
- 결제 적립금 발생 시: `적립금액 × 결제 가중치` 합산
- 포인트 사용/환전 시: **변동 없음**
- 적립금 사용 시: **변동 없음**

## 가중치 설정

관리자가 설정하는 가중치로 결제 고객과 활동 회원의 레벨업 속도를 조절합니다.

| 설정 키 | 기본값 | 설명 |
|---------|--------|------|
| `point_weight_payment` | 3 | 결제 적립금 → 누적포인트 변환 가중치 |
| `point_weight_activity` | 1 | 활동 포인트 → 누적포인트 변환 가중치 |

**예시: 가중치 효과 비교**

활동 가중치=1, 결제 가중치=3 일 때:

| 행동 | 실제 적립 | 누적포인트 합산 |
|------|----------|---------------|
| 글 작성 → 500pt | 500pt | 500 × 1 = **500** |
| 리뷰+사진 → 1,000pt | 1,000pt | 1,000 × 1 = **1,000** |
| 5만원 결제 → 1,500원 적립 | 1,500원 | 1,500 × 3 = **4,500** |
| 8만원 결제 → 2,400원 적립 | 2,400원 | 2,400 × 3 = **7,200** |

→ 결제 고객이 활동 회원보다 **3배 빠르게** 레벨업

## 환전 시스템

포인트를 적립금으로 변환하는 기능입니다.

| 설정 | 설명 | 기본값 |
|------|------|--------|
| `point_exchange_enabled` | 환전 활성화 여부 | N |
| `point_exchange_rate` | 1포인트당 환전 금액 | 1 (1pt = 1원) |
| `point_exchange_unit` | 환전 최소 단위 | 1,000pt |
| `point_exchange_min` | 환전 신청 최소 보유 포인트 | 1,000pt |

**환전 예시:**

설정: 환전 비율 1pt=1원, 환전 단위 1,000pt

| 단계 | point | balance | total_accumulated |
|------|-------|---------|-------------------|
| 환전 전 | 2,500pt | 1,000원 | 10,000 |
| 2,000pt 환전 신청 | -2,000pt → 500pt | +2,000원 → 3,000원 | 10,000 (변동 없음) |

**핵심: 환전은 누적포인트에 영향을 주지 않습니다.**

## 레벨 시스템

누적포인트(total_accumulated) 기준으로 레벨이 결정됩니다.

**레벨 테이블 예시: `Math.pow(l, 2) * 90`**

| 레벨 | 필요 누적포인트 | 도달 예시 |
|------|---------------|----------|
| 1 | 90 | 글 1개 작성 |
| 2 | 360 | 글 3개 + 댓글 5개 |
| 3 | 810 | 소규모 활동 |
| 5 | 2,250 | 1회 결제(5만원, 적립1,500×3=4,500) |
| 10 | 9,000 | 2~3회 결제 |
| 15 | 20,250 | 단골 고객 (5~7회 결제) |
| 20 | 36,000 | VIP 고객 |
| 30 | 81,000 | 최상위 고객 |

---

## 그룹 연동

특정 레벨에 도달하면 회원 그룹(등급)이 자동 변경됩니다.

| 레벨 | 그룹 | 혜택 예시 |
|------|------|----------|
| 1 | 일반 | 적립률 1% |
| 5 | 실버 | 적립률 3% |
| 15 | 골드 | 적립률 5%, 지명비 면제 |
| 25 | VIP | 적립률 7%, 우선 예약, 전용 쿠폰 |

**그룹 변경 방식:**
- 교체: 이전 그룹 제거 후 새 그룹 부여
- 추가: 이전 그룹 유지하며 새 그룹 추가

**누적포인트 기준이므로 레벨은 절대 내려가지 않습니다.**

---

## 도배 방지

게시판 활동이 포인트로 연결되므로 악용 방지가 필수입니다.

| 방지 수단 | 설정 위치 | 설명 |
|-----------|----------|------|
| 시간 제한 | 포인트 설정 "작성 후 N일까지" | 오래된 글에는 포인트 미지급 |
| 삭제시 회수 | 포인트 설정 "삭제시 회수" | 글 삭제 시 적립 포인트 환수 |
| 비추천 차감 | 포인트 설정 "비추천받음" | 비추천받으면 포인트 차감 |
| 최소 글자수 | 게시판 설정 `doc_length_limit` | 짧은 글 차단 |
| 일일 상한 | 포인트 설정 (향후) | 하루 최대 N건까지만 적립 |
| 공지 제외 | 포인트 설정 "공지 제외" | 공지글 읽기에 포인트 미지급 |

## DB 구조

### rzx_member_points

```sql
user_id           CHAR(36) PK     -- 회원 ID
point             INT DEFAULT 0   -- 사용 가능 포인트 (활동 적립, 환전 차감)
balance           INT DEFAULT 0   -- 적립금 잔액 (결제 적립, 환전 충전, 사용 차감)
total_accumulated INT DEFAULT 0   -- 누적 포인트 (레벨 기준, 증가만)
level             INT DEFAULT 1   -- 현재 레벨
updated_at        DATETIME        -- 최종 수정일
```

### rzx_point_levels

```sql
level     INT PK         -- 레벨 번호
point     INT DEFAULT 0  -- 필요 누적포인트
group_id  INT NULL       -- 연동 회원 그룹
```

### rzx_point_transactions

```sql
id          CHAR(36) PK
user_id     CHAR(36)
type        ENUM('earn','use','cancel','expire','admin_add','admin_deduct','refund','exchange')
amount      DECIMAL(12,2)
balance_after DECIMAL(12,2)
source      VARCHAR(50)    -- 'payment','board_write','board_comment','exchange' 등
source_id   CHAR(36)
description VARCHAR(255)
expires_at  TIMESTAMP NULL
admin_id    CHAR(36) NULL
created_at  TIMESTAMP
```

### rzx_settings (포인트 관련 키)

```
point_module_enabled      Y/N
point_name                포인트 이름
point_max_level           최고 레벨 (기본 30)
point_weight_payment      결제 가중치 (기본 3)
point_weight_activity     활동 가중치 (기본 1)
point_exchange_enabled    환전 활성화 Y/N
point_exchange_rate       환전 비율 (1pt = N원)
point_exchange_unit       환전 최소 단위
point_exchange_min        환전 최소 보유 포인트
point_signup              회원가입 포인트
point_login               로그인 포인트
point_insert_document     글 작성 포인트
...
```
