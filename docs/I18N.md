# 다국어 시스템 (I18n)

RezlyX의 다국어(국제화) 시스템 문서입니다.

## 지원 언어

기본 제공 언어:

| 코드 | 언어 | 네이티브명 |
|------|------|------------|
| `ko` | 한국어 | 한국어 |
| `en` | English | English |
| `ja` | 日本語 | 日本語 |
| `zh_CN` | 중국어(간체) | 中文(中国) |
| `zh_TW` | 중국어(번체) | 中文(臺灣) |
| `de` | 독일어 | Deutsch |
| `es` | 스페인어 | Español |
| `fr` | 프랑스어 | Français |
| `mn` | 몽골어 | Монгол |
| `ru` | 러시아어 | Русский |
| `tr` | 터키어 | Türkçe |
| `vi` | 베트남어 | Tiếng Việt |
| `id` | 인도네시아어 | Bahasa Indonesia |

> **참고**: 관리자 페이지에서 커스텀 언어를 추가할 수 있습니다.

## 사용법

### 초기화

```php
use RzxLib\Core\I18n\Translator;

// 번역 파일 경로와 함께 초기화
Translator::init(__DIR__ . '/resources/lang');

// 특정 로케일로 초기화
Translator::init(__DIR__ . '/resources/lang', 'ko');
```

### 번역 가져오기

```php
// 헬퍼 함수 사용
echo __('common.nav.home');           // "홈"
echo __('booking.title');              // "예약하기"

// 변수 치환
echo __('common.pagination.showing', [
    'from' => 1,
    'to' => 10,
    'total' => 100
]);  // "100개 중 1 - 10 표시"

// 마이페이지 환영 메시지
echo __('mypage.welcome', ['name' => '홍길동']);  // "홍길동님, 환영합니다!"
```

### 로케일 변경

```php
use RzxLib\Core\I18n\Translator;

// 현재 로케일 확인
echo Translator::getLocale();  // "ko"

// 로케일 변경 (세션/쿠키에 저장됨)
Translator::setLocale('en');

// 지원 로케일 목록
$locales = Translator::getSupportedLocales();  // ['ko', 'en', 'ja']

// 로케일 정보 (드롭다운용)
$allLocales = Translator::getAllLocales();
// [
//     ['code' => 'ko', 'name' => '한국어', 'active' => true],
//     ['code' => 'en', 'name' => 'English', 'active' => false],
//     ['code' => 'ja', 'name' => '日本語', 'active' => false],
// ]
```

### 복수형 처리

```php
// 번역 파일에서
// 'items' => '아이템 없음|:count개 아이템|:count개 아이템들'

echo trans_choice('common.items', 0);   // "아이템 없음"
echo trans_choice('common.items', 1);   // "1개 아이템"
echo trans_choice('common.items', 5);   // "5개 아이템들"
```

## 로케일 자동 감지

시스템은 다음 순서로 로케일을 감지합니다:

1. **세션** (`$_SESSION['locale']`)
2. **쿠키** (`$_COOKIE['locale']`)
3. **브라우저** (`Accept-Language` 헤더) - 자동 감지가 활성화된 경우만
4. **기본값** (DB 설정 또는 `ko`)

> **참고**: 언어 자동 감지는 관리자 설정에서 활성화/비활성화할 수 있습니다.

## DB 기반 언어 설정

### 개요

Translator 클래스는 `rzx_settings` 테이블에서 언어 설정을 로드합니다. 이를 통해 관리자 페이지에서 동적으로 언어 설정을 변경할 수 있습니다.

### 설정 키

| 키 | 설명 | 예시 값 |
|----|------|---------|
| `supported_languages` | 지원 언어 목록 (JSON) | `["ko","en","ja"]` |
| `default_language` | 기본 언어 코드 | `ko` |
| `language_auto_detect` | 자동 감지 활성화 | `1` 또는 `0` |
| `custom_languages` | 커스텀 언어 정보 (JSON) | `{"th":{"name":"ไทย","native":"ไทย"}}` |

### Translator 클래스 주요 메서드

```php
use RzxLib\Core\I18n\Translator;

// 전체 언어 정보 (기본 + 커스텀)
$allLanguages = Translator::getAllLanguages();
// ['ko' => ['name' => '한국어', 'native' => '한국어'], ...]

// 지원 언어 목록
$supported = Translator::getSupportedLocales();
// ['ko', 'en', 'ja']

// 자동 감지 활성화 여부
$autoDetect = Translator::isAutoDetect();
// true 또는 false

// 폴백(기본) 로케일
$fallback = Translator::getFallbackLocale();
// 'ko'

// 번역 파일 존재 여부 확인
$hasFiles = Translator::hasLanguageFiles('ko');
// true 또는 false

// 특정 언어의 번역 파일 목록
$files = Translator::getLanguageFiles('ko');
// ['common' => ['path' => '...', 'size' => 1234, 'modified' => 12345678], ...]

// 전체 번역 그룹 목록
$groups = Translator::getTranslationGroups();
// ['common', 'booking', 'admin', ...]

// 설정 다시 로드 (캐시 무효화)
Translator::reload();
```

## 관리자 언어 설정

### 접속 경로

`/admin/settings/language`

### 기능

1. **언어 자동 선택**: 브라우저 언어 감지 활성화/비활성화
2. **지원 언어 선택**: 사이트에서 사용할 언어 체크박스 선택
3. **기본 언어 선택**: 자동 감지 비활성화 시 또는 지원하지 않는 언어일 때 사용
4. **커스텀 언어 추가**: 기본 제공되지 않는 언어 추가
   - 언어 코드 (예: `th`, `ar`, `zh_HK`)
   - 네이티브 이름 (예: `ไทย`, `العربية`)
5. **커스텀 언어 삭제**: 추가한 언어 삭제 (기본 제공 언어는 삭제 불가)

### 파일 위치

- 뷰: `resources/views/admin/settings/language.php`
- 라우트: `/admin/settings/language` (GET/POST)

## 관리자 번역 관리

### 접속 경로

`/admin/settings/translations`

### 기능

1. **번역 파일 상태 확인**: 각 지원 언어별 번역 파일 존재 여부 표시
2. **번역 파일 생성**: 원본 언어를 선택하여 새 언어의 번역 파일 생성
3. **번역 편집**: 각 번역 파일을 클릭하여 키-값 편집
   - 원본 언어(한국어)와 대상 언어를 나란히 비교하며 편집
   - 중첩 배열 키 지원 (예: `auth.login.title`)

### 파일 위치

- 뷰: `resources/views/admin/settings/translations.php`
- 라우트: `/admin/settings/translations` (GET/POST)

### 번역 파일 생성 프로세스

```
1. 지원 언어 중 번역 파일이 없는 언어 확인
2. "파일 생성" 버튼 클릭
3. 원본 언어 선택 (보통 한국어)
4. 원본 언어 폴더의 모든 .php 파일을 대상 언어 폴더로 복사
5. 복사된 파일을 편집하여 번역 완성
```

## 새 언어 추가

### 방법 1: 관리자 페이지 사용 (권장)

1. **언어 설정** (`/admin/settings/language`)에서 "언어 추가" 클릭
2. 언어 코드 입력 (예: `th`, `ar`, `zh_HK`)
3. 네이티브 이름 입력 (예: `ไทย`, `العربية`)
4. 지원 언어 체크박스에서 새 언어 선택
5. **번역 관리** (`/admin/settings/translations`)에서 "파일 생성" 클릭
6. 원본 언어 선택 후 파일 생성
7. 각 번역 파일 클릭하여 번역 편집

### 방법 2: 수동 추가

#### 1. 번역 파일 생성

```bash
mkdir resources/lang/zh
cp resources/lang/ko/*.php resources/lang/zh/
```

#### 2. 번역 수정

```php
// resources/lang/zh/common.php
return [
    'site_name' => 'RezlyX',
    'nav' => [
        'home' => '首页',
        'booking' => '预订',
    ],
];
```

#### 3. DB에 언어 등록

```sql
-- 커스텀 언어 추가
UPDATE rzx_settings
SET `value` = JSON_SET(`value`, '$.zh', JSON_OBJECT('name', '中文', 'native', '中文', 'builtin', false))
WHERE `key` = 'custom_languages';

-- 지원 언어에 추가
UPDATE rzx_settings
SET `value` = JSON_ARRAY_APPEND(`value`, '$', 'zh')
WHERE `key` = 'supported_languages';
```

## DB 기반 다국어 번역

파일 기반 번역 외에 DB에 저장되는 사용자 정의 번역을 지원합니다.

### 테이블 구조

```sql
CREATE TABLE `rzx_translations` (
    `id` INT AUTO_INCREMENT NOT NULL,
    `lang_key` VARCHAR(255) NOT NULL,    -- 번역 키 (예: 'site.name')
    `locale` VARCHAR(10) NOT NULL,        -- 언어 코드 (ko, en, ja)
    `content` TEXT NOT NULL,              -- 번역 내용
    `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    `updated_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_key_locale` (`lang_key`, `locale`)
);
```

### 헬퍼 함수

| 함수 | 설명 |
|------|------|
| `db_trans($key, $locale, $default)` | DB에서 번역 가져오기 |
| `get_site_name($locale)` | 사이트 이름 (다국어) |
| `get_site_tagline($locale)` | 사이트 제목 (다국어) |
| `get_setting($key, $default)` | 설정값 가져오기 |

### 사용법

```php
// DB 번역 가져오기
$tagline = db_trans('site.tagline', 'ko', '기본값');

// 사이트 이름 (우선순위: DB 번역 > 설정 > 환경변수)

// 사이트 제목 (우선순위: DB 번역 > 설정)
$siteTagline = get_site_tagline();
```

### 우선순위

1. `rzx_translations` 테이블 (다국어 모달에서 입력한 값)
2. `rzx_settings` 테이블 (입력창에 직접 입력한 값 - 모든 언어 공통)
3. 환경변수 또는 기본값

### 번역 키 예시

| 키 | 설명 |
|----|------|
| `site.name` | 사이트 이름 |

## 네이티브 국가명 (Endonym) 시스템

전화번호 입력 컴포넌트에서는 번역 파일 대신 네이티브 국가명을 사용합니다.

### 개념

각 국가명을 해당 국가의 공식 언어로 표시하여 번역 파일 유지보수 부담을 제거합니다.

| 국가 | 네이티브명 | 설명 |
|------|-----------|------|
| 🇰🇷 한국 | 대한민국 | 한국어 |
| 🇯🇵 일본 | 日本 | 일본어 |
| 🇩🇪 독일 | Deutschland | 독일어 |
| 🇪🇸 스페인 | España | 스페인어 |
| 🇷🇺 러시아 | Россия | 러시아어 |
| 🇸🇦 사우디 | السعودية | 아랍어 |
| 🇮🇱 이스라엘 | Israel | 히브리어/영어 |

### 구현 방식

```php
// resources/views/components/phone-input.php
$countries = [
    ['+82', 'kr', '대한민국'],
    ['+81', 'jp', '日本'],
    ['+49', 'de', 'Deutschland'],
    // ...
];

// 사용
foreach ($countries as $country) {
    $code = $country[0];     // '+82'
    $iso = $country[1];      // 'kr'
    $name = $country[2];     // '대한민국'
}
```

### 장점

- **번역 불필요**: 180개+ 국가의 번역 파일 유지보수 부담 없음
- **정확성**: 각 국가의 공식 명칭 사용
- **인식성**: 국기 이모지와 함께 표시되어 낯선 문자도 식별 가능
- **일관성**: 사용자 언어 설정과 무관하게 동일한 표시

### 기존 번역 키와의 관계

`common.countries.*` 키는 레거시 호환성을 위해 유지되지만, 전화번호 컴포넌트에서는 사용하지 않습니다.

## 다국어 날짜 포맷팅

### 개요

RezlyX의 예약 시스템에서 날짜를 다국어로 표시하기 위해 `formatReservationDate()` 함수를 제공합니다. 이 함수는 ISO 날짜 문자열을 받아 각 언어별로 적절하게 포맷된 문자열을 반환합니다.

### 지원 언어 및 포맷

| 언어 | 코드 | 형식 예 | 비고 |
|------|------|---------|------|
| 한국어 | `ko` | `2026년 3월 29일 (일)` | 요일 한글 표기 |
| 영어 | `en` | `Sunday, March 29, 2026` | 월 영문 표기 |
| 일본어 | `ja` | `2026年3月29日(日)` | 요일 일문 표기 |
| 중국어(간체) | `zh_CN` | `2026年3月29日(日)` | 월 한자, 요일 원형 |
| 중국어(번체) | `zh_TW` | `2026年3月29日(日)` | 월 한자, 요일 원형 |
| 독일어 | `de` | `Sonntag, 29. März 2026` | 요일/월 독문 표기 |
| 스페인어 | `es` | `domingo, 29 de marzo de 2026` | 요일/월 소문자 시작 |
| 프랑스어 | `fr` | `dimanche 29 mars 2026` | 요일/월 소문자 시작 |
| 인도네시아어 | `id` | `Minggu, 29 Maret 2026` | 요일/월 대문자 시작 |
| 몽골어 | `mn` | `2026.03.29` | ISO 형식 + 요일명 |
| 러시아어 | `ru` | `воскресенье, 29 марта 2026` | 요일/월 소문자 시작 |
| 터키어 | `tr` | `Pazar, 29 Mart 2026` | 요일/월 대문자 시작 |
| 베트남어 | `vi` | `Chủ nhật, 29 tháng 3, 2026` | 요일/월 대문자 시작 |

### 사용 예시

#### 기본 사용

```php
// 현재 로케일로 포맷
echo formatReservationDate('2026-03-29', $currentLocale);

// 특정 로케일로 포맷
echo formatReservationDate('2026-03-29', 'ja');  // "2026年3月29日(日)"
echo formatReservationDate('2026-03-29', 'en');  // "Sunday, March 29, 2026"
```

#### 예약 상세 페이지 통합

```php
<!-- resources/views/customer/booking/detail.php -->
<div class="reservation-info">
    <dl>
        <dt><?= __('booking.detail.date') ?></dt>
        <dd><?= formatReservationDate($reservation['reservation_date'], $currentLocale) ?></dd>
    </dl>
    <dl>
        <dt><?= __('booking.detail.time') ?></dt>
        <dd><?= $reservation['reservation_time'] ?></dd>
    </dl>
</div>
```

### 함수 파라미터

```php
function formatReservationDate($dateString, $locale = 'ko')
```

| 파라미터 | 타입 | 설명 | 기본값 |
|---------|------|------|--------|
| `$dateString` | string | ISO 형식 날짜 (`YYYY-MM-DD`) 또는 PHP `strtotime()` 호환 형식 | 필수 |
| `$locale` | string | 언어 코드 (ISO 639-1 또는 ISO 639-3) | `'ko'` |

### 반환값

- **성공**: 포맷된 날짜 문자열 (언어별로 다름)
- **실패**: 빈 문자열 (`''`) - `strtotime()`이 날짜를 파싱하지 못한 경우

### 함수 구조

함수 내부는 다음과 같이 동작합니다:

1. **입력 파싱**: `strtotime()`으로 날짜를 타임스탬프로 변환
2. **날짜 정보 추출**: 년, 월, 일, 요일(0~6) 추출
3. **로케일별 포맷**: `switch` 문으로 언어에 맞게 포맷팅
4. **폴백**: 지원하지 않는 로케일은 기본값(`'en'`)으로 처리

### 언어 추가 (확장)

새로운 언어를 지원하려면 `detail.php`의 `formatReservationDate()` 함수에 `case` 블록을 추가합니다:

```php
case 'th':  // 태국어
    return (int)$day . ' ' . ['', 'มกราคม', 'กุมภาพันธ์', ...][($month-1)] . ' ' . ($year + 543);
    // 불교력 사용 (2026 → 2569)
```

## 스킨 템플릿에서의 번역 처리

### 회원 스킨 번역 우선순위

회원 스킨(로그인, 회원가입 등)에서는 다음 순서로 번역이 적용됩니다:

1. **MemberSkinLoader**에서 전달된 `$translations`
2. **스킨 템플릿 내부**에서 로케일 감지 후 병합

### 로케일 감지 순서

```php
$_skinLocale = 'ko';
if (!empty($_SESSION['locale'])) {
    $_skinLocale = $_SESSION['locale'];
} elseif (!empty($_COOKIE['locale'])) {
    $_skinLocale = $_COOKIE['locale'];
} elseif (function_exists('current_locale')) {
    $_skinLocale = current_locale();
}
```

### 주의사항: 변수 스코프 충돌

PHP의 `include` 문은 포함된 파일과 변수 스코프를 공유합니다. 컴포넌트 파일에서 `$translations` 변수를 덮어쓰면 부모 템플릿의 번역이 손실됩니다.

**잘못된 예시** (image_cropper.php):
```php
// 부모 템플릿의 $translations를 덮어씀
$translations = $cropperConfig['translations'] ?? [];
```

**올바른 예시**:
```php
// 별도 변수명 사용으로 충돌 방지
$_cropperTranslations = $cropperConfig['translations'] ?? [];
```

### 컴포넌트 개발 가이드라인

1. **변수명 접두사 사용**: 컴포넌트 내부 변수에는 `$_` 접두사 사용
2. **전역 변수 보존**: `$translations`, `$config` 등 전역 변수 덮어쓰기 금지
3. **배열 병합 시 주의**: `array_merge()` 사용 시 원본 배열 유지

## 공용 언어 선택기 컴포넌트

### 개요

`resources/views/components/language-selector.php`는 admin-topbar와 동일한 UI의 공용 언어 드롭다운입니다. LanguageModule을 사용하여 DB 활성 언어를 표시합니다.

### 사용법

어느 페이지에서든 한 줄로 include:

```php
<?php include BASE_PATH . '/resources/views/components/language-selector.php'; ?>
```

### 필요 조건

- `$siteSettings` 변수가 스코프에 있어야 함 (DB rzx_settings 배열)
- `BASE_PATH` 상수가 정의되어 있어야 함

### LanguageModule

`rzxlib/Core/Modules/LanguageModule.php`는 언어 데이터의 중앙 모듈입니다.

```php
use RzxLib\Core\Modules\LanguageModule;

$langData = LanguageModule::getData($siteSettings, $currentLocale);
// 반환값:
// [
//   'languages' => ['ko' => '한국어', 'en' => 'English', ...],
//   'allLanguages' => ['ko' => ['name' => '한국어', 'native' => '한국어'], ...],
//   'supportedCodes' => ['ko', 'en', 'ja', ...],
//   'currentLocale' => 'ko',
//   'defaultLocale' => 'ko',
//   'currentLangInfo' => ['name' => '한국어', 'native' => '한국어'],
// ]
```

### 사용되는 위치

| 위치 | 방식 |
|------|------|
| admin-topbar.php | 자체 구현 (레퍼런스 표준) |
| skins/default/components/header.php | 공용 컴포넌트 include |
