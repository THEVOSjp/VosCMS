# 선불제 (프리페이드) 시스템

## 1. 개요

한국에서 유행하는 미용실 선불제 시스템을 RezlyX Salon에 도입하기 위한 기획 문서.
매장의 현금 흐름 안정화, 고객 이탈 방지, 객단가 상승을 목표로 한다.

---

## 2. 선불제 유형

### 2.1 충전형 (Prepaid Balance)

일정 금액을 미리 충전하고 시술 시 잔액에서 차감하는 방식.

```
고객 마이페이지:
  잔액: ¥50,000
  [충전하기]

충전 플랜:
  ¥10,000 충전 → ¥10,000 (0% 보너스)
  ¥30,000 충전 → ¥33,000 (10% 보너스)
  ¥50,000 충전 → ¥57,500 (15% 보너스)
  ¥100,000 충전 → ¥120,000 (20% 보너스)

시술 시:
  커트 ¥6,000 → 잔액에서 차감 → 잔액 ¥44,000
```

**특징:**
- 충전 금액이 클수록 보너스율 증가 → 대량 충전 유도
- 모든 서비스에 사용 가능 (범용)
- 잔액이 남으면 계속 방문 → 고객 이탈 방지

### 2.2 회원권형 (Service Pass)

특정 서비스를 횟수 기반으로 패키지 구매하는 방식.

```
커트 10회권: ¥64,000 (정가 ¥80,000, 20% 할인)
컬러 5회권: ¥40,000 (정가 ¥50,000, 20% 할인)
풀케어 세트 3회권: ¥45,000

시술 시:
  커트 1회 사용 → 남은 횟수 9회
```

**특징:**
- 서비스별로 횟수를 묶어 판매
- 할인율을 매장이 자유롭게 설정
- 번들(세트) 서비스와 연계 가능

### 2.3 정액제 (Subscription)

월정액으로 일정 횟수 또는 무제한 이용하는 방식.

```
Basic: ¥9,800/월 — 커트 월 1회
Standard: ¥15,000/월 — 커트 무제한
Premium: ¥25,000/월 — 커트 + 컬러 무제한

자동 결제 (매월)
```

**특징:**
- 고객 입장: 월 비용 예측 가능
- 매장 입장: 안정적인 반복 수익
- PG사 정기결제(サブスク) 연동 필요

### 2.4 선결제 할인 (Next Visit Prepay)

시술 완료 후 다음 방문을 미리 예약+결제하면 할인하는 방식.

```
시술 완료 후 POS에서:
  "다음 방문 예약 + 선결제하시면 10% 할인"
  다음 커트 ¥6,000 → ¥5,400 선결제
  → 예약 확정 + 결제 완료
```

**특징:**
- 구현이 가장 간단
- 노쇼 방지에 효과적
- 다음 방문 보장 → 재방문율 상승

## 3. 비즈니스 효과

### 매장 입장

| 효과 | 설명 |
|------|------|
| **현금 흐름 안정** | 미래 매출을 미리 확보 |
| **고객 이탈 방지** | 잔액/횟수가 남으면 계속 방문 |
| **객단가 상승** | 한 번에 큰 금액 결제 유도 |
| **노쇼 감소** | 선결제 고객은 노쇼율이 극히 낮음 |
| **예측 가능한 매출** | 정액제의 경우 월 매출 예측 가능 |

### 고객 입장

| 효과 | 설명 |
|------|------|
| **할인 혜택** | 충전/회원권 구매 시 할인 |
| **결제 편의** | 매번 결제 불필요, 잔액에서 자동 차감 |
| **예산 관리** | 정액제의 경우 월 비용 확정 |

## 4. DB 설계 (안)

### 4.1 충전형

```sql
-- 충전 플랜 (관리자가 설정)
rzx_prepaid_plans
  id, name, amount, bonus_amount, bonus_rate, is_active, sort_order

-- 고객별 잔액
rzx_prepaid_balances
  id, user_id, balance, total_charged, total_used, created_at, updated_at

-- 충전/사용 이력
rzx_prepaid_history
  id, user_id, type(charge/use/refund), amount, balance_after,
  reference_type(reservation/manual), reference_id,
  payment_method, description, created_at
```

### 4.2 회원권형

```sql
-- 회원권 상품 (관리자가 설정)
rzx_service_passes
  id, name, service_id, bundle_id, total_count, price,
  original_price, discount_rate, validity_days, is_active, sort_order

-- 고객별 구매한 회원권
rzx_customer_passes
  id, user_id, pass_id, remaining_count, purchased_at,
  expires_at, status(active/expired/used_up), payment_id

-- 사용 이력
rzx_pass_usage
  id, customer_pass_id, reservation_id, used_at, staff_id
```

### 4.3 정액제

```sql
-- 정액 플랜
rzx_subscription_plans
  id, name, monthly_price, services(JSON), max_count_per_month,
  is_unlimited, is_active

-- 고객 구독
rzx_subscriptions
  id, user_id, plan_id, status(active/paused/cancelled),
  started_at, next_billing_at, cancelled_at, payment_method

-- 월별 사용 이력
rzx_subscription_usage
  id, subscription_id, reservation_id, month, used_at
```

### 4.4 선결제 할인

```sql
-- 기존 rzx_reservations 테이블에 컬럼 추가
  prepaid_amount DECIMAL(10,2) DEFAULT 0    -- 선결제 금액
  prepaid_discount_rate DECIMAL(5,2) DEFAULT 0  -- 할인율

-- 기존 rzx_payments 테이블 활용
  type: 'prepay_next_visit'
```

## 5. POS 연동

### 5.1 결제 화면 흐름

```
POS 결제 시:
  시술비 ¥10,000
  
  [결제 방법 선택]
  ├── 현금
  ├── 카드
  ├── QR 결제
  ├── ★ 프리페이드 잔액 (¥50,000) ← 신규
  └── ★ 회원권 사용 (커트 남은 9회) ← 신규

  프리페이드 선택 시:
    ¥10,000 차감 → 잔액 ¥40,000

  회원권 선택 시:
    커트 1회 사용 → 남은 8회
    (추가 서비스 ¥3,000은 별도 결제)
```

### 5.2 복합 결제

```
시술비 ¥15,000
  프리페이드 잔액: ¥5,000 (잔액 전부 사용)
  + 카드: ¥10,000
  = 합계 ¥15,000
```

## 7. 관리자 기능

### 7.1 충전 플랜 관리

- 플랜 등록/편집 (금액, 보너스율)
- 플랜 활성/비활성
- 충전 통계 (총 충전액, 미사용 잔액 합계)

### 7.2 회원권 관리

- 회원권 상품 등록 (서비스 연동, 횟수, 가격, 유효기간)
- 판매/사용 현황
- 만료 임박 고객 알림

### 7.3 리포트

| 리포트 | 내용 |
|--------|------|
| 충전 현황 | 기간별 충전 총액, 건수, 평균 충전액 |
| 미사용 잔액 | 전체 고객 미사용 잔액 합계 (부채) |
| 회원권 판매 | 기간별 판매 건수, 매출 |
| 회원권 소진율 | 평균 사용률, 만료 폐기율 |
| 정액제 현황 | 활성 구독 수, MRR(월 반복 매출) |

## 8. 법적 고려사항

### 일본

- **資金決済法 (자금결제법):** 선불식 지불 수단(前払式支払手段)에 해당할 수 있음
  - 잔액 미사용분이 ¥1,000만 초과 시 공탁금(供託金) 의무
  - 자가형(自家型): 발행자 매장에서만 사용 → 신고 의무
  - 제3자형: 타 매장에서도 사용 → 등록 의무 (해당 없음)
- **소비자 보호:** 환불 정책 명시, 유효기간 명시 필요
- **특정商取引法:** 정액제(サブスク)의 경우 해약 조건 명시 필요

### 한국

- **선불전자지급수단:** 전자금융거래법 적용 가능
- 소규모 매장 자체 충전은 "자가발행"으로 비교적 규제 완화

## 10. 구현 우선순위

| 순서 | 유형 | 이유 |
|------|------|------|
| **1** | 충전형 (Prepaid Balance) | 가장 범용적, 기존 포인트 시스템과 유사한 구조 |
| **2** | 회원권형 (Service Pass) | 번들 서비스와 연계 가능, 충전형 위에 확장 |
| **3** | 선결제 할인 (Next Visit Prepay) | POS에 간단 추가, 기존 예약금 구조 활용 |
| **4** | 정액제 (Subscription) | PG 정기결제 연동 필요, 복잡도 높음 |
